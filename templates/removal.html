Firmware Removal Procedures

Standard Removal

  1. Device enters Download Mode automatically
  2. PIT is read and validated
  3. BL (Bootloader) partition is preserved
  4. AP (System) partition is flashed with clean ROM
  5. CP (Modem) is preserved if compatible
  6. HOME_CSC is used to preserve user data
  7. Device reboots into clean system

Clean Removal

  1. Device enters Download Mode
  2. Full PIT read and validation
  3. All partitions except BL are wiped
  4. AP flashed with stock firmware
  5. CP flashed with stock modem
  6. CSC (not HOME_CSC) used — full factory reset
  7. Userdata and cache formatted

Emergency Recovery

  1. Force device into Download Mode (Vol Down + Power)
  2. PIT file is loaded (extracted from device or provided)
  3. Repartition flag is set — all partitions rebuilt
  4. BL + AP + CP + CSC all flashed sequentially
  5. Post-flash: device may take 5-10 min for first boot
  6. If boot loop: repeat with different CSC variant

Bloatware Removal

  1. Device in normal Android mode with USB debugging ON
  2. ADB connection established
  3. Package list is enumerated: adb shell pm list packages
  4. Bloatware packages identified by CSC/carrier patterns
  5. Packages disabled (not uninstalled): pm disable-user
  6. Device reboot changes are verified
